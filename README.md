# Blog Image Maker

오렌지 블로그 포스트의 썸네일 이미지를 자동 생성하는 내부 도구.

---

## 기획 배경

블로그 포스트를 발행할 때마다 썸네일 이미지를 직접 디자인하는 데 드는 시간을 줄이기 위해 시작했다. 단순히 이미지를 생성하는 것에서 나아가, 오렌지 브랜드 스타일(오렌지 포인트 컬러, 검정 배경, 아이콘 일러스트 스타일)을 일관되게 유지하면서도 글의 내용을 반영한 이미지가 자동으로 나오는 것이 목표였다.

핵심 과제는 두 가지였다:
1. AI가 생성한 이미지가 매번 브랜드 스타일을 벗어나지 않도록 고정하는 것
2. 글마다 다른 아이콘과 구도가 나오도록 다양성을 확보하는 것

---

## 전체 작동 흐름

```
사용자 입력 (제목 + 본문)
        ↓
[Step 1] GPT-4o — CONCEPT_SYSTEM 프롬프트
         · 레이아웃 랜덤 선택 후 user 메시지에 고정 주입
         · 글의 핵심 내용에서 아이콘 추출
         · 아이콘 이름, 위치, 크기, 연결 요소를 한국어 60-80자로 기술
         → 아이콘 배치 프롬프트 반환
        ↓
사용자가 프롬프트 확인 및 수동 수정 가능
        ↓
[Step 2] GPT-image-1 — STYLE_RULES를 프롬프트에 append
         · 순수 검정 배경 (#000000)
         · 메인 아이콘 focalColor solid fill, 나머지는 흰 outline만
         · 장식 요소 (점, +, 다이아몬드, 짧은 선) 크고 작게 산재
         → 1024×1024 PNG (검정 배경)
        ↓
[Sharp] 검정 배경 제거
         · √(r²+g²+b²) < 40 인 픽셀을 투명화
         → 투명 배경 PNG
        ↓
[Puppeteer] HTML 합성
         · 2400×1260px 캔버스에 제목 + 일러스트 + 로고 배치
         · Pretendard Bold 폰트, Figma 좌표 기반 레이아웃
         → 최종 썸네일 PNG (output/ 폴더에 저장)
```

---

## 기술 스택

| 기술 | 역할 | 선택 이유 |
|------|------|-----------|
| **Node.js + Express** | 서버 | 간단한 API 서버 구성에 최적 |
| **GPT-4o** | 아이콘 배치 프롬프트 생성 | 한국어 이해도 및 JSON 출력 안정성 |
| **GPT-image-1** | 일러스트 이미지 생성 | 스타일 지시 이행률이 가장 높음 |
| **Sharp** | 검정 배경 제거 | 빠른 픽셀 단위 처리 |
| **Puppeteer** | HTML → PNG 합성 | 웹 기술(CSS, SVG, 폰트)을 그대로 활용 가능 |
| **Pretendard** | 한국어 타이포그래피 | 웨이트별 자간 품질이 우수한 한국어 폰트 |

## 핵심 컴포넌트

### CONCEPT_SYSTEM (GPT-4o 시스템 프롬프트)

GPT-4o가 아이콘 배치 프롬프트를 생성할 때 사용하는 지시문. 역할은 "무엇을 그릴까"를 결정하는 것으로, 스타일이나 색상은 다루지 않는다.

핵심 규칙:
- 메인 아이콘 1개 + 보조 아이콘 1-2개 선택
- 아이콘은 해당 글의 내용에서 직접 추출한 구체적 사물이어야 함 (글마다 달라야 함)
- 레이아웃은 시스템이 랜덤으로 지정해 user 메시지에 주입 (모델이 선택하지 않음)
- 메인 아이콘과 보조 아이콘 사이에 시각적 연결 요소(점선, 화살표 등) 명시
- 출력은 JSON `{ "prompt": "..." }` 단일 필드

few-shot 예시 2개 포함 (좌우 대비형, 흐름형) — 규칙 나열보다 예시가 출력 형식을 더 효과적으로 고정하기 때문.

### STYLE_RULES (이미지 생성 스타일 고정)

이미지 모델에 전달할 최종 프롬프트에 항상 append되는 고정 스타일 규칙. 역할은 "어떻게 그릴까"를 결정하는 것.

핵심 규칙:
- 순수 검정 배경 (#000000)
- 메인 아이콘은 focalColor(오렌지 또는 파란색) solid fill
- 나머지 요소는 흰색 outline만 (fill 없음)
- 요소들 사이 점선, 점선 호, 또는 화살표로 연결
- 장식 요소: 점, +, 다이아몬드, 짧은 선을 크고 작게 혼합
- 텍스트, 숫자, 레이블, 현실적 렌더링, 3D, 그라디언트 금지

### LAYOUTS (랜덤 레이아웃 시스템)

6개의 레이아웃을 사전 정의하고 요청마다 랜덤으로 하나를 선택해 GPT-4o에게 강제 주입한다. 아이콘 구도가 항상 동일한 패턴(중앙+주변)으로 수렴하는 문제를 해결하기 위해 도입.

```
좌우 대비형  /  흐름형  /  클로즈업형
상하 구조형  /  대각선형  /  좌편향형
```

### focalColor (포인트 컬러 자동 결정)

배경색의 색조(Hue)와 채도, 밝기를 분석해 오렌지 계열 배경이면 파란색(#0075FF), 그 외에는 오렌지(#FF6F1F)를 포인트 컬러로 사용한다. 오렌지 배경에 오렌지 아이콘이 묻히는 문제를 방지하기 위함.

### 검정 배경 제거 (Sharp)

이미지 모델에 순수 검정 배경으로 생성하도록 지시한 뒤, `√(r²+g²+b²) < 40` 조건으로 픽셀을 투명화한다. GPT-image-1은 지시를 충실히 따라 배경이 거의 순수 (0,0,0)으로 나오기 때문에 이 방식이 안정적으로 동작한다.

### HTML 합성 (Puppeteer)

Figma로 설계한 레이아웃 좌표를 기반으로 HTML/CSS를 작성하고, Puppeteer가 2400×1260px로 스크린샷을 찍는다. 실제 표시 크기(1200×630px)의 2배로 렌더링해 레티나 디스플레이에서도 선명하게 보이도록 했다.

레이아웃 좌표 (Figma 원본 × 2):
- 제목: left 200px / top 152px / width 1094px
- 일러스트: left 1340px / top 180px / width 860px / height 900px
- 로고: left 200px / top 982px / width 600px

## API

| 엔드포인트 | 메서드 | 역할 |
|-----------|--------|------|
| `/api/prompt` | POST | GPT-4o로 아이콘 배치 프롬프트 생성 |
| `/api/illustration` | POST | 이미지 모델로 일러스트 생성 |
| `/api/render` | POST | Puppeteer로 최종 썸네일 합성 |

## 실행 방법 (로컬)

이 앱은 클라우드에 배포하지 않고 각자 로컬에서 실행한다. 내부 도구 특성상 사용 인원이 적고, 이 앱은 Puppeteer(헤드리스 Chrome)를 사용하기 때문에 클라우드 배포 시 Chrome 의존성 설정이 별도로 필요해 오히려 번거롭다. 로컬 실행이 가장 안정적이다.

### 1단계 — Node.js 설치

https://nodejs.org 에 접속해 **LTS** 버전을 다운로드 후 설치한다. (이미 설치되어 있으면 건너뜀)

### 2단계 — 프로젝트 받기

GitHub에서 zip으로 다운로드하거나 아래 명령어로 클론한다.

```bash
git clone 레포지토리_주소
cd orange-blog
```

### 3단계 — .env 파일 생성

프로젝트 폴더 최상단에 `.env` 파일을 만들고 아래 내용을 입력한다. API 키는 팀에서 공유받는다.

```
OPENAI_API_KEY=sk-...
GEMINI_API_KEY=...
```

### 4단계 — 패키지 설치

```bash
npm install
```

### 5단계 — 서버 실행

```bash
npm start
```

터미널에 `서버 시작: http://localhost:3000` 메시지가 뜨면 준비 완료.

### 6단계 — 브라우저 접속

```
http://localhost:3000
```

---

> 사용할 때마다 `npm start`로 서버를 켜야 한다. 터미널을 닫으면 서버도 종료된다.
