# Design Doc — Blog Image Maker

## 개요

블로그 포스트 제목과 본문을 입력하면 브랜드 스타일의 썸네일 이미지를 자동 생성하는 도구.
최종 출력은 2400×1260px PNG (표시 크기 1200×630px).

---

## 시스템 구조

```
[제목 + 본문]
     ↓
[GPT-4o] CONCEPT_SYSTEM 프롬프트
     → 아이콘 배치 프롬프트 (한국어, 60-80자)
     ↓
[GPT-image-1] STYLE_RULES 추가 append
     → 일러스트 PNG (1024×1024, 검정 배경)
     ↓
[Sharp] 검정 배경 제거 → 투명 PNG
     ↓
[Puppeteer] HTML 합성 (제목 + 일러스트 + 로고)
     → 최종 썸네일 PNG
```

---

## 핵심 의사결정

### 1. 왜 프롬프트 생성과 일러스트 생성을 분리했나

처음엔 한 번에 처리하려 했으나, 이미지 모델에 직접 "어떤 아이콘을 그릴지"까지 맡기면 결과가 매번 달라지고 제어가 어려웠다. 프롬프트 생성 단계를 분리하면:

- 사용자가 프롬프트를 보고 수동으로 수정할 수 있다
- "프롬프트 재생성"으로 다른 구도를 시도할 수 있다
- 이미지 모델을 교체해도 프롬프트 로직은 유지된다

### 2. CONCEPT_SYSTEM과 STYLE_RULES를 왜 분리했나

역할이 다르다.

- **CONCEPT_SYSTEM** — "무엇을 그릴까" (아이콘 선택, 배치 구도)
- **STYLE_RULES** — "어떻게 그릴까" (색상, 선굵기, 배경, 금지 요소)

CONCEPT_SYSTEM은 GPT-4o에게 전달해 프롬프트를 생성할 때 쓰고, STYLE_RULES는 이미지 모델에 전달할 최종 프롬프트에 append한다. 이렇게 분리하면 스타일 규칙을 수정해도 CONCEPT_SYSTEM을 건드릴 필요가 없다.

### 3. 레이아웃 랜덤화

초기에는 아이콘 구도가 항상 "중앙에 메인, 주변에 보조 3개" 패턴으로 수렴하는 문제가 있었다. 이를 해결하기 위해 6가지 레이아웃을 사전 정의하고 요청마다 랜덤으로 하나를 선택해 user 메시지에 고정으로 주입한다.

```
좌우 대비형 / 흐름형 / 클로즈업형 / 상하 구조형 / 대각선형 / 좌편향형
```

레이아웃은 GPT-4o가 선택하는 게 아니라 시스템이 강제 지정하기 때문에, 모델이 특정 구도에 편향되는 현상이 줄어든다.

### 4. Few-shot 예시를 CONCEPT_SYSTEM에 추가한 이유

규칙을 아무리 늘려도 GPT-4o가 서사적/해석적 프롬프트를 생성하는 문제가 반복됐다. 예를 들어 "AI와 인간의 협업을 상징하는..." 같은 추상적 표현. 금지 규칙보다 올바른 출력 형식의 예시를 직접 보여주는 게 훨씬 효과적이었다.

예시는 두 개로, 각각 다른 레이아웃을 사용해 "레이아웃은 input에서 주어지는 것"임을 학습시킨다.

### 5. focalColor — 배경에 따라 포인트 컬러 결정

배경이 오렌지 계열이면 파란색(#0075FF)을, 그 외에는 오렌지(#FF6F1F)를 포인트 컬러로 사용한다. 오렌지 배경에 오렌지 아이콘이 묻히는 문제를 방지하기 위함.

### 6. 검정 배경 제거 방식

이미지 모델에 "순수 검정 배경으로 그려라"고 지시한 뒤, Sharp로 `√(r²+g²+b²) < 40` 인 픽셀을 투명화한다. GPT-image-1은 지시를 충실히 따라 배경이 거의 순수 검정(0,0,0)으로 나오기 때문에 이 방식이 잘 동작한다.

### 7. Gemini를 이미지 생성 모델에서 제외한 이유

초기에는 Gemini(`gemini-3-pro-image-preview`)를 선택지로 제공했으나 실제 테스트에서 두 가지 문제가 확인됐다.

1. **지시 이행률 낮음** — STYLE_RULES에 "순수 검정 배경(#000000)"을 명시해도 완전한 검정이 아닌 어두운 회색이나 다른 색으로 배경을 생성하는 경우가 많았다.
2. **배경 제거 실패** — 결과적으로 `√(r²+g²+b²) < 40` 기준을 통과하지 못하는 픽셀이 남아 배경이 제거되지 않고 합성 결과물에 검은 사각형이 그대로 노출됐다.

GPT-image-1 대비 이 스타일 지시에 대한 이행 신뢰도가 낮다고 판단해 옵션에서 제거했다.

### 7. 해상도 2× 렌더링

Puppeteer로 HTML을 캡처할 때 2400×1260px로 렌더링한 뒤 배포 시 절반 크기(1200×630px)로 표시한다. 레티나 디스플레이에서 선명하게 보이게 하기 위함.

---

## 현재 한계 및 개선 여지

- 프롬프트 재생성을 여러 번 해도 비슷한 레이아웃이 뽑힐 수 있음 (랜덤이므로)
- 일러스트 스타일이 아이콘 종류에 따라 품질 편차 있음
- 배경색이 다양해질 경우 focalColor 로직 정교화 필요
